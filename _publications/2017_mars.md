We present a formal model for fragmentation and reassembly protocols running on top of the standardised CAN bus, which is widely used in automotive and aerospace applications.

Although the CAN bus already provides a built-in prioritisation mechanism, we argue that this mechanism alone is insufficient in practice. To address this limitation, we introduce an additional protocol layer that improves message handling and prioritisation behaviour. The paper provides a rigorous formal specification of the resulting CAN bus protocol stack.
